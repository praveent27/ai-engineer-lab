Where to Run Notebooks for Free

You don't need a GPU laptop for weeks 1–8. For deep learning (weeks 9+), these platforms give you free GPU access.

Google Colab ★ Recommended

Zero setup, runs in browser, free T4 GPU. Best for quick experiments and weeks 1–12 content. Mount Google Drive for persistent storage.

GPU: T4 (15GB) · RAM: 12GB · Session: ~12h · Free

Kaggle Notebooks FREE

30h/week GPU quota, P100 GPU, built-in datasets. Great for competitions and the EDA projects. Background execution supported.

GPU: P100 (16GB) · RAM: 30GB · Quota: 30h/wk · Free

AWS SageMaker Studio Lab FREE

Persistent storage (no re-upload), JupyterLab interface, good GitHub integration. 4h GPU sessions. No credit card needed.

GPU: T4 · RAM: 16GB · Session: 4h · Free (signup)

Lightning.ai ★ Best for DL

Best UX for serious deep learning work. Persistent home drive, team collab, PyTorch Lightning native. Free tier with GPU credits.

GPU: Various · Persistent · Free tier + paid

Paperspace Gradient FREE

Free GPU notebooks (M4000, RTX4000), persistent storage, no session timeouts. Now part of DigitalOcean.

GPU: M4000/RTX4000 · Persistent · Free tier

Azure ML Free Tier FREE

Azure Machine Learning offers a free tier with limited compute. Best for weeks 17+ when you need Azure AI Services anyway. Spin up notebooks directly in the Azure portal.

GPU: Configurable · Best for Azure AI labs

💡 My Recommendation

Weeks 1–8: Google Colab (CPU is fine — no GPU needed for NumPy, Pandas, scikit-learn). Weeks 9–16: Google Colab with T4 GPU for PyTorch and deep learning. Weeks 17–28: Azure free tier for AI Services labs + Colab for any model training.

How to Use This Resource

The Study Process

A systematic workflow for each week. Consistency beats intensity — 1.5 hours of focused practice beats 4 hours of passive reading.

1

Read the Week Summary First

Each week page starts with a summary of what you'll learn and why it matters. Read this before touching any code. Understand the "why" before the "how".

2

Open the Notebook in Colab

Click the "Open in Colab" badge on each notebook. Or clone the repo: git clone https://github.com/praveent27/ai-engineer-lab.git and run locally with jupyter lab.

3

Run Every Cell — Don't Just Read

Execute each cell, read the output, then modify it. Change parameters, break things, add print statements. Active experimentation beats passive reading 10x.

4

Complete the Exercises Without Looking

Every notebook has exercises at the end. Try them before checking the solutions. If you get stuck for more than 10 minutes, peek at the solution, then close it and try again.

5

Review the Cheat Sheet

Each week has a syntax quick-reference. Print it or keep it open in a second tab. Use it during exercises, then try to recall from memory.

6

Push Your Work to GitHub

After finishing each day's notebook, commit to your fork. This builds your portfolio and creates a habit of documentation. Your completed notebooks ARE your proof of learning.

7

Friday = Rest. Sunday = Build.

Fridays are off. Saturdays are deep-dive theory days. Sundays are project days — apply everything from the week on a real dataset or mini-project.
