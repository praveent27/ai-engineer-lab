Introduction to ML

1.5 hours

Why this matters: Before building models, you need to understand what ML actually is. Supervised learning predicts known outputs. Unsupervised learning finds hidden patterns. This foundation determines which algorithms you choose for which problems.

📓 Day1_Introduction_to_ML.ipynb

🤖 What is Machine Learning?

ML is learning patterns from data to make predictions or decisions without explicit programming. Instead of writing rules like "if height > 180cm then basketball player", ML learns the relationship automatically from examples.

📋 Supervised Learning

Training data has labels/targets. Regression predicts continuous values (house prices). Classification predicts categories (spam/not spam). 80% of real-world ML is supervised.

🔍 Unsupervised Learning

No labels — algorithm finds hidden structure. Clustering groups similar items. Dimensionality reduction compresses data. Used for customer segmentation, anomaly detection, and preprocessing.

🎯 ML Problem Types

Regression: predict numbers. Classification: predict categories. Clustering: group data. Recommendation: suggest items. Generation: create content. Each needs different evaluation metrics.

ML PROBLEM TYPES

Supervised→Labeled data, prediction tasks
Regression→Predict continuous values
Classification→Predict discrete categories
Unsupervised→Unlabeled data, pattern discovery
Clustering→Group similar items

KEY TAKEAWAYS

  • ML learns from data, not code. The algorithm finds patterns humans might miss.
  • Supervised = teacher (labeled data). Unsupervised = exploration (no labels).
  • Most business problems are supervised: predict sales, detect fraud, recommend products.
  • Choose the algorithm based on your data type and problem type, not the other way around.
  • ML is probabilistic — models give confidence scores, not certainties.
TUE

Bias-Variance Tradeoff

1.5 hours

Why this matters: Every ML model has errors. Bias is systematic errors (too simple). Variance is sensitivity to training data (too complex). Understanding this tradeoff is how you tune models and choose algorithms.

📓 Day2_Bias_Variance_Tradeoff.ipynb

🎯 Total Error = Bias² + Variance + Irreducible Error

Bias: how far predictions are from true values (systematic error). Variance: how much predictions vary with different training data. Irreducible: noise in data (can't fix).

📉 Underfitting (High Bias)

Model too simple — doesn't capture patterns. Linear model on quadratic data. High training error, high validation error. Solution: more complex model, more features, reduce regularisation.

📈 Overfitting (High Variance)

Model too complex — memorises training data. Perfect training accuracy, poor validation. Solution: simpler model, regularisation, more data, cross-validation.

⚖️ The Sweet Spot

Balance bias and variance for minimum total error. Use validation curves to find optimal complexity. Early stopping, dropout, and ensemble methods help.

KEY TAKEAWAYS

  • High bias = underfitting = model too simple. Fix: increase complexity.
  • High variance = overfitting = model too complex. Fix: simplify or regularise.
  • Training error decreases as model complexity increases. Validation error forms U-shape.
  • More data reduces variance. Better features reduce bias.
  • Cross-validation estimates true model performance on unseen data.
WED

Model Evaluation Metrics

1.5 hours

Why this matters: Accuracy is misleading. For imbalanced data or different costs, you need the right metric. Precision vs recall tradeoff. ROC curves vs precision-recall curves. This determines if your model is actually useful.

📓 Day3_Model_Evaluation_Metrics.ipynb

📊 Confusion Matrix

True Positive, False Positive, True Negative, False Negative. Foundation for all classification metrics. Always plot this first — accuracy hides problems.

🎯 Precision vs Recall

Precision: of predicted positives, how many correct? (TP/(TP+FP)). Recall: of actual positives, how many found? (TP/(TP+FN)). F1 = harmonic mean. Choose based on business cost.

📈 ROC & AUC

ROC plots TPR vs FPR at different thresholds. AUC measures discriminative ability. Good for balanced classes. Use precision-recall for imbalanced data.

📏 Regression Metrics

MAE: mean absolute error (robust to outliers). MSE: mean squared error (penalises large errors). R²: explained variance (0–1, higher better).

KEY TAKEAWAYS

  • Accuracy fails on imbalanced data. Use confusion matrix always.
  • Precision important when false positives costly (spam filter). Recall when false negatives costly (medical diagnosis).
  • F1 balances precision and recall. Use when both matter equally.
  • AUC good for ranking. Use when you care about prediction quality across thresholds.
  • Cross-validation gives reliable estimates. Never evaluate on training data.
THU

Linear Regression from Scratch

2 hours

Why this matters: Linear regression is the foundation of all ML. Understanding gradient descent, loss functions, and optimisation from scratch makes every algorithm make sense. No black boxes.

📓 Day4_Linear_Regression_from_Scratch.ipynb

📈 The Model: ŷ = w·x + b

Find weights w and bias b that minimise the difference between predictions ŷ and true values y. Simple but powerful — basis for neural networks.

📉 Loss Function: MSE

Mean Squared Error: (1/n)Σ(y - ŷ)². Penalises large errors more. Differentiable — key for gradient descent.

⬇️ Gradient Descent

Iteratively update weights: w = w - α·∂L/∂w. Learning rate α controls step size. Too small = slow. Too large = overshoot minimum.

🔄 Normal Equation

Closed-form solution: w = (XᵀX)⁻¹Xᵀy. No iteration needed. Works for small datasets. Basis for analytical solutions.

KEY TAKEAWAYS

  • Linear regression assumes linear relationship. Check with scatter plots.
  • Gradient descent: compute gradients, update parameters, repeat.
  • Feature scaling important for convergence. Standardise inputs.
  • Regularisation (L1/L2) prevents overfitting by penalising large weights.
  • Linear models interpretable — coefficients show feature importance.

SAT

Logistic Regression from Scratch

2 hours

Why this matters: Classification is 80% of ML problems. Logistic regression is the simplest classifier and foundation for neural networks. Understanding sigmoid, cross-entropy, and decision boundaries is crucial.

📓 Day5_Logistic_Regression_from_Scratch.ipynb

📊 The Sigmoid Function

σ(z) = 1/(1+e⁻ᶻ). Squashes any input to (0,1) range. Interpret as probability. Derivative: σ'(z) = σ(z)·(1-σ(z)).

🎯 Binary Cross-Entropy Loss

L = -[y·log(ŷ) + (1-y)·log(1-ŷ)]. Penalises confident wrong predictions heavily. Differentiable for gradient descent.

⚖️ Decision Boundary

ŷ ≥ 0.5 → class 1. Linear boundary: w·x + b = 0. Can be non-linear with feature engineering (polynomial features).

🔍 Multi-class Extension

Softmax instead of sigmoid. Cross-entropy generalises. One-vs-rest or multinomial approaches.

KEY TAKEAWAYS

  • Logistic regression outputs probabilities, not hard classifications.
  • Sigmoid maps linear predictions to probabilities. Cross-entropy measures how well probabilities match true labels.
  • Decision threshold (0.5 default) can be tuned based on precision/recall needs.
  • Assumes linear decision boundary. Use polynomial features for non-linear problems.
  • Interpretable — coefficients show log-odds change per unit feature increase.

01 ML Types What is the difference between supervised and unsupervised learning? ▼
Supervised: training data has labels/targets. Algorithm learns to predict known outputs (regression/classification). Unsupervised: no labels. Algorithm finds hidden patterns (clustering/dimensionality reduction). Supervised is like having a teacher; unsupervised is exploration.
02 ML Types Give examples of regression vs classification problems. ▼
Regression: predict continuous numbers — house prices, temperature, stock prices. Classification: predict discrete categories — spam/not spam, digit recognition (0-9), medical diagnosis (sick/healthy). Regression outputs numbers; classification outputs categories.
03 Bias-Variance Explain underfitting vs overfitting in one sentence each. ▼
Underfitting: model too simple to capture data patterns (high bias). Overfitting: model memorises training data noise instead of generalising (high variance). Underfitting misses the signal; overfitting learns the noise.
04 Bias-Variance Your model has perfect training accuracy but poor validation accuracy. What's happening? ▼
Overfitting. The model has memorised the training data (low bias on train) but can't generalise to new data (high variance). Training error low, validation error high. Solution: simplify model, add regularisation, get more data, or use cross-validation.
05 Bias-Variance How does cross-validation help with overfitting? ▼
Cross-validation trains on multiple train/val splits, giving a more reliable estimate of true generalisation performance. It prevents overfitting to a single validation set and helps detect if your model is truly learning patterns vs memorising specific data points.
06 Metrics When would you use precision vs recall as your primary metric? ▼
Precision: when false positives are costly — spam filter (don't want to mark legit emails as spam). Recall: when false negatives are costly — medical diagnosis (don't want to miss sick patients). Precision = quality of positive predictions. Recall = completeness of positive predictions.
07 Metrics Why is accuracy misleading for imbalanced datasets? ▼
A model predicting "always majority class" can have high accuracy but be useless. Example: 95% accuracy on fraud detection (where fraud is 5%) by predicting "no fraud" for everything. Accuracy hides the fact that the model fails on the minority class. Use precision, recall, F1, or AUC instead.
08 Metrics What does AUC-ROC measure and when should you use it? ▼
AUC measures the model's ability to discriminate between classes across all classification thresholds. Higher AUC = better ranking of positive vs negative examples. Use when you care about prediction quality regardless of threshold, or for comparing models. Good for balanced classes; use precision-recall curves for imbalanced data.
09 Linear Regression What is the normal equation for linear regression? ▼
w = (XᵀX)⁻¹Xᵀy — closed-form solution that finds optimal weights in one step. No iteration needed. Works for small datasets. Requires matrix inversion, so not suitable for very large datasets (use gradient descent instead).
10 Linear Regression How does gradient descent update the weights in linear regression? ▼
w = w - α·∂L/∂w where ∂L/∂w = -2·Xᵀ·(y - ŷ) (for MSE loss). The gradient points in the direction of steepest increase in loss, so we subtract it to decrease loss. Learning rate α controls step size — too small = slow convergence, too large = overshoot minimum.
11 Linear Regression Why do we need to scale features for gradient descent? ▼
Gradient descent converges faster when features have similar scales. If one feature ranges 0-1 and another 0-1000, the gradient will be dominated by the large-scale feature, causing slow or unstable convergence. Standardisation (mean=0, std=1) or min-max scaling ensures all features contribute equally to the gradient.
12 Logistic Regression What is the sigmoid function and why do we use it? ▼
σ(z) = 1/(1+e⁻ᶻ) — squashes any real number input to (0,1) range, perfect for probabilities. Logistic regression uses it to turn linear predictions into class probabilities. The output can be interpreted as P(class=1), and we classify as class 1 when P ≥ 0.5.
13 Logistic Regression Why use cross-entropy loss instead of MSE for classification? ▼
MSE treats classification as regression, penalising wrong predictions linearly. Cross-entropy -[y·log(ŷ)+(1-y)·log(1-ŷ)] penalises confident wrong predictions exponentially more. For example, predicting 0.01 when true label is 1 hurts much more than predicting 0.4. This matches probabilistic interpretation and leads to better gradients.
14 Logistic Regression How do you handle multi-class classification with logistic regression? ▼
One-vs-Rest: train K binary classifiers (one for each class vs all others). Multinomial: use softmax to output probabilities summing to 1, generalise cross-entropy. Softmax preferred when classes are mutually exclusive. One-vs-rest simpler but can have calibration issues.
15 ML Pipeline What are the steps in a typical ML pipeline? ▼
1. Data collection & exploration. 2. Preprocessing (clean, scale, encode). 3. Train/val/test split. 4. Model training. 5. Validation & tuning. 6. Final evaluation on test set. 7. Deployment & monitoring. Never peek at test data during development.
16 ML Pipeline Why split data into train/validation/test sets? ▼
Train: fit model parameters. Validation: tune hyperparameters, select models. Test: unbiased final evaluation. Prevents overfitting to validation set. Test set simulates real-world performance on unseen data. Never use test data for model development decisions.
17 Regularisation What is L1 vs L2 regularisation? ▼
17
Regularisation What is L1 vs L2 regularisation? ▼
L1 (Lasso): penalises sum of absolute weights |w|. Drives weights to exactly zero → feature selection. Use for sparse models. L2 (Ridge): penalises sum of squared weights w². Shrinks weights toward zero but rarely to zero → prevents overfitting. Use when all features potentially useful.
18 Regularisation How does regularisation prevent overfitting? ▼
Regularisation adds a penalty term to the loss function that discourages complex models. L2 shrinks weights, making the model less sensitive to individual training examples. L1 creates sparse models by zeroing irrelevant features. This forces the model to learn simpler, more generalisable patterns instead of memorising noise.
19 Model Selection How do you choose between different ML algorithms for a problem? ▼
1. Understand data: size, type, distribution, missing values. 2. Problem type: regression/classification/clustering. 3. Try baselines: simple models first (linear regression, logistic regression). 4. Compare performance: use cross-validation, appropriate metrics. 5. Consider tradeoffs: interpretability vs accuracy, training time vs prediction time.
20 Model Selection What is hyperparameter tuning and why is it important? ▼
Hyperparameters are model settings not learned from data (learning rate, regularisation strength, tree depth). Tuning finds optimal values using validation performance. Important because default hyperparameters often suboptimal — proper tuning can significantly improve model performance. Use grid search, random search, or Bayesian optimisation.
