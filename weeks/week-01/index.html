MON

NumPy Fundamentals

1.5 hours

Why this matters: NumPy is the bedrock of every ML library in Python. PyTorch tensors, scikit-learn matrices, Pandas DataFrames — they're all NumPy arrays underneath. If you understand NumPy, you understand the data structures of ML.

📓 Day1_NumPy_Fundamentals.ipynb

🧱 Arrays vs Lists

NumPy arrays are fixed-type, contiguous memory blocks. Operations are vectorised — no Python loops needed. np.array([1,2,3]) creates a vector. np.zeros((3,4)) creates a matrix. Always use np.random.seed(42) for reproducibility.

📐 Broadcasting

NumPy automatically aligns arrays of different shapes for element-wise operations. This is how you normalise columns in one line: (data - data.mean(axis=0)) / data.std(axis=0). Rules: dimensions are compared right-to-left, size-1 dimensions stretch.

⚡ Vectorised Operations

Replace Python loops with NumPy operations for 50–100x speedups. a + b, a @ b, np.where(), boolean masking (arr[arr > 5]). If you're writing for i in range(len(arr)), there's almost always a NumPy way.

🧠 ML Connection: Forward Pass

A neural network layer is literally output = input @ weights + bias followed by an activation function. Matrix multiplication (@) IS the core computation. Cosine similarity (dot(a,b) / (|a|*|b|)) is how vector search works in RAG.

KEY TAKEAWAYS

  • axis=0 collapses rows (operates DOWN columns). axis=1 collapses columns (operates ACROSS rows).
  • reshape(-1) flattens. reshape(2,-1) auto-calculates the missing dimension.
  • np.linalg.norm(v) gives vector magnitude — used in distance calculations and normalisation.
  • Boolean indexing (arr[arr > 5]) is how you filter data without loops — the ML workhorse.
  • Always set np.random.seed() at the top of every notebook for reproducible results.
TUE

Pandas Deep Dive

1.5 hours

Why this matters: Pandas is how you load, clean, transform, and explore real data before feeding it to any ML model. Every Kaggle kernel starts with Pandas. Every data pipeline ends with a Pandas-like operation.

📓 Day2_Pandas_DeepDive.ipynb

📊 DataFrame = SQL Table

df[df['col'] > 5] = WHERE. df.groupby('col').mean() = GROUP BY. pd.merge(df1, df2, on='key') = JOIN. .transform('mean') = window function. Your SQL instincts transfer directly.

🔗 Method Chaining

Chain operations for clean, readable pipelines: df.assign(new=...).query('x > 5').groupby('col').agg(...).sort_values(...). One expression, no intermediate variables.

🩹 Missing Data

df.isnull().sum() to count. fillna(median()) for numeric. fillna(mode()[0]) for categorical. dropna(subset=['critical_col']) when you can't impute. Always check missing % first.

📁 I/O: Parquet > CSV

df.to_parquet('file.parquet') is faster, smaller, and preserves dtypes. Use CSV for sharing, Parquet for processing — you already know this from your data engineering background.

PANDAS ↔ SQL MAPPING

SELECT col1, col2→df[['col1', 'col2']]
WHERE col > 5→df[df['col'] > 5]
GROUP BY col→df.groupby('col')
ORDER BY col DESC→df.sort_values('col', ascending=False)
JOIN ON key→pd.merge(df1, df2, on='key')
WINDOW FUNC→.groupby().transform()
CASE WHEN→np.where() or .apply()

KEY TAKEAWAYS

  • .loc[] = label-based, .iloc[] = integer-position-based. Use .loc by default.
  • .query("col > 5") is cleaner than df[df['col'] > 5] for complex filters.
  • .apply(func) for custom logic. Lambda for one-liners: .apply(lambda x: x.upper()).
  • Check df.dtypes early — wrong types (string instead of numeric) cause silent bugs.

Matplotlib & Seaborn

1.5 hours

Why this matters: Visualisation is how you debug data AND models. Confusion matrices, loss curves, feature importance plots, and correlation heatmaps are your daily tools as an ML practitioner.

📓 Day3_Matplotlib_Seaborn.ipynb

📈 Matplotlib = Full Control

Use the fig, ax = plt.subplots() pattern — never plt.plot() directly. Subplots give you full control. Always: ax.set_title(), ax.set_xlabel(), ax.legend(), plt.tight_layout(), plt.savefig('plot.png', dpi=150).

🎨 Seaborn = Statistical Plots

sns.heatmap(df.corr()) for correlation. sns.pairplot(df, hue='target') for all relationships at once. sns.boxplot(x='cat', y='num') for distributions by group. Seaborn wraps Matplotlib with beautiful defaults.

🤖 ML-Specific Charts

Training loss curves (train vs val — spot overfitting), confusion matrices (model prediction errors), feature importance (horizontal bar charts), ROC curves. You'll build all of these during the journey.

KEY TAKEAWAYS

  • Always use fig, ax = plt.subplots(figsize=(10,6)) — not plt.plot() directly.
  • sns.heatmap(corr, annot=True, cmap='RdBu_r', center=0) — your go-to for correlation matrices.
  • When mean ≫ median in a distribution → right-skewed data → consider log transform.
  • Pair plots (sns.pairplot) should be your first visualisation on any new dataset.
THU

Linear Algebra for ML

1.5 hours

Why this matters: Neural network layers are matrix multiplications. Embeddings are vectors. PCA is eigenvalue decomposition. Cosine similarity powers Azure AI Search. Understanding these operations makes every ML concept click.

📓 Day4_Linear_Algebra_for_ML.ipynb

→ Dot Product = Similarity Measure

np.dot(a, b) — positive means similar direction, zero means perpendicular, negative means opposite. This is literally how attention works in transformers and how vector search finds relevant documents.

× Matrix Multiplication = Neural Layer

output = X @ W + b — that's a neural network layer. X is input data (samples × features), W is weights (features × neurons), output is (samples × neurons). Shape rule: (m×n) @ (n×p) → (m×p).

λ Eigenvalues = PCA

Eigenvectors of the covariance matrix = principal components (directions of maximum variance). Eigenvalues = how much variance each component explains. PCA is dimensionality reduction via eigendecomposition.

‖ Norms = Distance & Regularisation

L2 norm = Euclidean distance (KNN). L1 norm = Manhattan distance (Lasso regularisation, drives weights to zero). L2 regularisation (Ridge) shrinks weights. These show up in model tuning constantly.

LINEAR ALGEBRA → ML MAPPING

Dot product→Attention, similarity search, layer computation
Cosine similarity→RAG retrieval, Azure AI Search, embeddings
Matrix multiply→Every neural network layer
Eigendecomposition→PCA, spectral clustering
L1 / L2 norm→Regularisation, distance metrics
Unit vector→Normalised embeddings

SAT

Statistics for ML

3 hours

Why this matters: Distributions underpin model assumptions. Bayes theorem is the foundation of probabilistic ML. Hypothesis testing is how you compare models. Entropy drives decision tree splits. Statistics is the "why" behind every ML algorithm.

📓 Day6_Statistics_for_ML.ipynb

📊 Descriptive Stats

Mean (sensitive to outliers) vs median (robust). When mean ≫ median = right skew. IQR for outlier detection: anything below Q1-1.5×IQR or above Q3+1.5×IQR is an outlier. Use df.describe() as your first check.

🔔 Normal Distribution

68% of data within 1σ, 95% within 2σ, 99.7% within 3σ. Central Limit Theorem: sample means are always normally distributed (even from non-normal populations) — this is why BatchNorm works in neural networks.

🧮 Bayes Theorem

P(A|B) = P(B|A) × P(A) / P(B) — "update your belief given evidence." This is exactly how Naive Bayes classifier works: given word "free" appears, what's the probability of spam? Foundational for probabilistic ML.

📏 Entropy & Information Gain

H = -Σ p × log₂(p) — measures uncertainty. High entropy = unpredictable. Decision trees split data to minimise entropy (maximise information gain). Cross-entropy loss in neural networks is based on this same concept.

KEY TAKEAWAYS

  • p-value < 0.05 → statistically significant difference. Used for comparing model performances and A/B testing.
  • Correlation ≠ causation. And Pearson r ≈ 0 doesn't mean no relationship — always check scatter plots for non-linear patterns.
  • Z-score normalisation: (x - mean) / std. Min-max: (x - min) / (max - min). Know when to use each.
  • Entropy drives decision tree splits. Cross-entropy is the loss function for classification in neural networks.

01 NumPy What is the difference between a Python list and a NumPy array? Why does it matter for ML? ▼
NumPy arrays are fixed-type and stored in contiguous memory, enabling vectorised operations (C-level loops instead of Python loops). This makes them 50–100x faster for numerical operations. ML libraries (scikit-learn, PyTorch) are built on this — every weight matrix, every batch of images, every embedding is a NumPy-style array.
02 NumPy What does axis=0 vs axis=1 mean in arr.sum(axis=?)? ▼
axis=0 collapses rows — operates down each column (gives column-wise results). axis=1 collapses columns — operates across each row (gives row-wise results). Think of it as: the axis you specify is the one that disappears.
03 NumPy Explain broadcasting. What happens when you do (data - data.mean(axis=0)) / data.std(axis=0)? ▼
Broadcasting automatically aligns arrays of different shapes. Here, data is (n, m) and data.mean(axis=0) is (m,). NumPy stretches the (m,) vector across all n rows, subtracting each column's mean from every row. The result is z-score normalisation — each column gets mean≈0, std≈1. This is standard feature scaling in ML.
04 NumPy Write the code for a neural network forward pass in NumPy (one layer). ▼
z = X @ W + b followed by a = sigmoid(z) where sigmoid = lambda x: 1/(1+np.exp(-x)). X is (samples × features), W is (features × neurons), b is (neurons,). The @ is matrix multiplication — the core computation of every neural network layer.
05 NumPy What is cosine similarity? How is it used in Azure AI Search? ▼
cos_sim = dot(a,b) / (|a| × |b|). It measures the angle between two vectors, ranging from -1 (opposite) to +1 (identical direction). In Azure AI Search vector search, documents and queries are converted to embedding vectors, and cosine similarity finds the most semantically similar documents — the core of RAG retrieval.
06 Pandas What is the Pandas equivalent of a SQL GROUP BY col HAVING count > 5? ▼
df.groupby('col').filter(lambda g: len(g) > 5) — this keeps entire groups. Or aggregate first then filter: result = df.groupby('col').agg(...) followed by result[result['count'] > 5]. The second approach is more common in practice.
07 Pandas What is .transform() and how does it differ from .agg()? ▼
.agg() reduces — returns one row per group (like SQL GROUP BY). .transform() broadcasts — returns a value for every original row, like a SQL window function. Example: df.groupby('dept')['salary'].transform('mean') gives each employee their department's average salary alongside their own row.
08 Pandas When should you use .loc[] vs .iloc[]? ▼
.loc[] is label-based — uses index labels and column names. .iloc[] is integer position-based — uses 0-indexed positions. Use .loc by default for clarity. Use .iloc when you need positional access (e.g., "first 5 rows" = df.iloc[:5]). Key gotcha: .loc slicing is inclusive on both ends; .iloc is exclusive on the end.
09 Pandas Why is Parquet better than CSV for data work? Name 3 reasons. ▼
1. Columnar storage — reads only the columns you need, much faster for analytics. 2. Preserves dtypes — no "is this column a string or int?" bugs on reload. 3. Compressed — typically 5–10x smaller than CSV. Bonus: supports schema evolution and is the native format for Spark, Delta Lake, and most modern data platforms.
10 Visualisation When should you use a box plot vs a histogram vs a violin plot? ▼
Histogram: see the full shape of one variable's distribution (skew, modes, gaps). Box plot: compare distributions across groups + spot outliers (IQR-based). Violin plot: like a box plot but also shows the distribution shape (KDE). Use histograms for EDA, box plots for group comparisons, violin plots when you need both.
11 Visualisation You plot a correlation heatmap and see r = 0.92 between two features. What do you do? ▼
r = 0.92 means high multicollinearity. These features carry nearly the same information. For linear models, you should drop one or use PCA to combine them — otherwise you get unstable coefficients. For tree-based models it matters less (they can handle it), but it still inflates feature importance for both. Always investigate: is one derived from the other?
12 Linear Algebra What does matrix multiplication (m×n) @ (n×p) actually compute? Why must the inner dimensions match? ▼
Each element in the result is a dot product of a row from the left matrix with a column from the right matrix. The inner dimensions (n) must match because that's the length of the vectors being dotted together. Result shape is (m×p). In a neural network: m = samples, n = input features, p = output neurons.
13 Linear Algebra What are eigenvalues and eigenvectors? How do they relate to PCA? ▼
For a matrix A, an eigenvector v satisfies A·v = λ·v — the matrix only scales the vector, doesn't change its direction. λ is the eigenvalue (the scale factor). In PCA: eigenvectors of the covariance matrix give the directions of maximum variance (principal components). Eigenvalues tell you how much variance each component explains. You keep the top-k components to reduce dimensions.
14 Linear Algebra What is the difference between L1 and L2 regularisation? When would you use each? ▼
L1 (Lasso): penalises sum of absolute weights. Drives some weights to exactly zero → feature selection. Use when you suspect many features are irrelevant. L2 (Ridge): penalises sum of squared weights. Shrinks all weights toward zero but rarely to zero → prevents overfitting without eliminating features. Use when all features are potentially useful. ElasticNet combines both.
15 Statistics Explain Bayes' theorem in one sentence, then give an ML example. ▼
"Update your prior belief about A after observing evidence B." P(A|B) = P(B|A)·P(A) / P(B). ML example: spam classifier — prior P(spam)=30%, if the word "free" appears in 80% of spam but only 10% of legit emails, then P(spam|"free") = (0.8×0.3)/(0.8×0.3 + 0.1×0.7) = 77.4%. This is exactly how Naive Bayes works.
16 Statistics What is the Central Limit Theorem and why does it matter for ML? ▼
The mean of sufficiently large samples (n≥30) from ANY distribution will be approximately normally distributed. This is why: batch statistics in BatchNorm work, confidence intervals are valid, t-tests can compare model performances, and many ML assumptions about "normally distributed errors" hold even when the raw data isn't normal.
17 Statistics Your dataset has Pearson r = 0.02 between two variables. Can you conclude there's no relationship? Why or why not? ▼
No! Pearson r only measures linear relationships. There could be a strong non-linear relationship (e.g., quadratic, U-shaped) that Pearson completely misses. Always plot a scatter chart. Use Spearman's rank correlation for monotonic non-linear relationships, or mutual information for any type of dependency.
18 Statistics What is entropy? Why do decision trees care about it? ▼
H = -Σ p·log₂(p) — measures uncertainty/disorder. High entropy = unpredictable (e.g., fair coin = 1 bit). Low entropy = predictable (biased coin). Decision trees split data to maximise information gain (= parent entropy minus weighted child entropy). Each split makes subsets more "pure" (lower entropy). Cross-entropy loss in neural networks is the same concept.
19 EDA You find a feature with 25% missing values. Walk through your decision process for handling it. ▼
1. Check if missingness is random or systematic (plot missing vs target). 2. If the feature is important (high correlation with target), impute — use median for numeric (robust to outliers) or mode for categorical. Consider group-wise imputation (e.g., median by class). 3. If unimportant or >50% missing, consider dropping. 4. Create a binary is_missing flag — the fact that data is missing can itself be predictive.
20 EDA Your target variable has 90% class 0 and 10% class 1. What problems does this cause, and what can you do? ▼
This is class imbalance. A model predicting "always 0" gets 90% accuracy but is useless. Problems: misleading accuracy, poor recall for the minority class. Solutions: 1. Use metrics that handle imbalance — F1, precision, recall, ROC-AUC instead of accuracy. 2. Stratified train/test splits. 3. Resampling — SMOTE (oversample minority) or undersample majority. 4. Class weights in the model. 5. Threshold tuning on prediction probabilities.
