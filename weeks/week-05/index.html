MON

Decision Trees

3 hours

Why this matters: Decision trees are the foundation of all ensemble methods (random forests, XGBoost, LightGBM). They're interpretable, handle mixed data types, and can capture non-linear patterns without feature scaling. Understanding tree growth and stopping criteria is essential for hyperparameter tuning.

📓 Day1_Decision_Trees.ipynb

🌳 Tree Growth via Information Gain

At each node, split on the feature that minimises Gini impurity (classification) or MSE (regression). Gini = 1 - Σp² where p is class probability. Lower Gini = purer node. Trees grow greedily — each split is locally optimal, not globally optimal.

📊 Stopping Criteria

max_depth = tree height. min_samples_split = minimum samples to split a node. min_samples_leaf = minimum samples in leaf node. max_features = features considered at each split. Shallow trees = high bias, deep trees = high variance (overfitting).

🔍 Feature Importance

Trees rank feature importance by how much they reduce Gini/MSE across all splits. High importance = feature used early in tree (splits many samples). This is your debugging tool — if important features are low-ranked, your data is messy or the pattern is weak.

⚖️ ML Connection: Tree Bias-Variance Trade-off

Shallow, pruned trees = high bias, low variance. Deep, unpruned trees = low bias, high variance. The sweet spot is found via cross-validation on max_depth and min_samples_leaf.

KEY TAKEAWAYS

  • Trees don't need feature scaling or encoding (handle categorical natively).
  • max_depth=3–7 is a good starting range for individual trees. Deeper = better fit, higher variance.
  • Feature importance from trees is relative, not absolute — always sanity-check with domain knowledge.
  • Trees are prone to overfitting — always validate on a held-out test set.
TUE

Ensemble Methods: Random Forests & Bagging

3 hours

Why this matters: Random forests reduce tree overfitting via averaging. Bootstrap samples + random feature subsets = uncorrelated trees. Random forests are fast, require minimal tuning, and are the go-to baseline for many practitioners. They scale to large datasets and handle imbalanced classes.

📓 Day2_Ensemble_Methods.ipynb

🎲 Bootstrap Aggregating (Bagging)

Sample n rows with replacement to create multiple training sets. Train independent trees on each. Average predictions (regression) or vote (classification). This reduces variance without increasing bias — because we're averaging uncorrelated learners. Key insight: each tree sees ~63% of the original data.

🌲 Random Forests = Bagging + Random Features

At each split, only consider sqrt(n_features) random features (classification) or n_features/3 (regression). This decorrelates trees further. Result: lower variance than bagging alone, and built-in feature importance ranking.

⚙️ Key Hyperparameters

n_estimators = number of trees (more = better, but with diminishing returns). max_depth = depth per tree (usually leave None or very deep). min_samples_leaf = minimum samples in leaf. n_jobs=-1 for parallel training.

📈 Out-of-Bag (OOB) Score

Each tree sees ~63% of data; evaluate on the held-out ~37%. OOB score ≈ CV score with zero training overhead. Quick way to estimate generalization without explicit validation split.

RANDOM FOREST INTUITION

Problem→Single tree overfits (high variance)
Solution 1→Average multiple trees (bagging)
Solution 2→Randomise features per split
Result→Low variance, low bias, fast

KEY TAKEAWAYS

  • Random forests are fast, parallel, and need little tuning — they're your first baseline.
  • n_estimators has diminishing returns past ~100–200 trees.
  • Feature importance from RF gives you a ranking of which features matter — use this for feature selection.
  • OOB score is essentially free CV; check that OOB ≈ test score (if not, data leakage or distribution shift).
WED

Gradient Boosting: XGBoost

3 hours

Why this matters: XGBoost powers 80% of Kaggle competitions. It's faster and more accurate than sklearn's GradientBoosting. It handles missing data, computes feature importance in multiple ways, and has regularisation built-in. Learning XGBoost is essential for your ML practitioner toolkit.

📓 Day3_XGBoost.ipynb

🚀 Gradient Boosting: Sequential Improvement

Each tree fits the residuals (errors) of the previous ensemble. New predictions = old predictions + (learning_rate × new tree). This sequential focus on errors is why boosting often outperforms bagging. Regularisation is critical — without it, boosting overfits.

🏆 XGBoost: Production-Ready Boosting

XGBoost is optimised for speed and memory via tree pruning, column sampling, and hardware acceleration. It includes L1/L2 regularisation, custom loss functions, and built-in cross-validation. xgb.train() uses early stopping to prevent overfitting.

🎯 Hyperparameter Tuning Strategy

Learning rate (eta): 0.01–0.3 (lower = slower but often better). Tree depth (max_depth): 3–8 (shallow trees work best in boosting). Regularisation (lambda, gamma): prevent overfitting. Early stopping: watch validation loss, stop if no improvement after N rounds.

📊 Feature Importance in XGBoost

Gain = improvement in accuracy. Cover = number of samples affected. Frequency = how often feature is used. Use Gain for feature selection; Frequency for interpretability.

KEY TAKEAWAYS

  • Start with max_depth=5, eta=0.1, n_estimators=100 and tune from there.
  • Early stopping is essential — let XGBoost find the optimal number of trees via validation set.
  • XGBoost handles missing values natively — no need for imputation.
  • Regularisation (L1/L2) is critical — without it, XGBoost overfits even faster than random forests.
  • scale_pos_weight for imbalanced classification; sample_weight for weighted training.
THU

Support Vector Machines (SVMs)

3 hours

Why this matters: SVMs are geometrically intuitive and theoretically elegant. They find the maximum-margin hyperplane — the decision boundary that maximises distance to the nearest points. With non-linear kernels, SVMs can fit complex patterns. They're less popular than tree-based methods in competition settings, but essential for understanding ML fundamentals.

📓 Day4_Support_Vector_Machines.ipynb

🎯 Maximum Margin Principle

SVM finds a hyperplane that maximises the distance (margin) to the nearest data points on either side. Points closest to the margin are support vectors. The margin's width is inversely related to model complexity — wider margins generalise better.

💥 Soft Margin & Regularisation (C)

Allow some misclassification (soft margin) controlled by penalty parameter C. High C = strict margins (low bias, high variance). Low C = looser margins (high bias, low variance). Always scale features to [-1, 1] — SVM is distance-based.

🔌 Kernels: Linear → Non-linear

Linear: works for linearly separable data. RBF (Radial Basis Function): handles non-linear patterns by mapping to higher dimensions. Polynomial: fits polynomial boundaries. Start with RBF; tune gamma (RBF kernel width).

⚠️ SVM Limitations

Slow on large datasets (O(n²) training). Hard to interpret (black box for non-linear kernels). Requires feature scaling. Less memory-efficient than tree-based methods. Use for small-to-medium datasets where interpretability matters.

SVM KERNEL SELECTION GUIDE

Linearly separable→kernel='linear'
Non-linear, small data→kernel='rbf', tune gamma
Polynomial decision boundary→kernel='poly', tune degree
Large dataset→Use linear or tree-based instead

KEY TAKEAWAYS

  • Always scale SVM inputs with StandardScaler — it's distance-based.
  • C=1.0 is default; tune on test set. gamma='scale' is a good default for RBF.
  • SVM is slower than trees on large data — use for interpretability or small datasets.
  • For multi-class, scikit-learn uses one-vs-rest strategy (slower than trees).

K-Nearest Neighbors & Distance Metrics

3 hours

Why this matters: KNN is simple but powerful for small datasets and non-parametric learning. It's also the foundation for many anomaly detection and recommendation systems. Understanding distance metrics (Euclidean, Manhattan, cosine) is essential for vector search and RAG pipelines.

📓 Day6_KNN_Distance_Metrics.ipynb

👥 KNN Algorithm

For prediction, find the K nearest training samples and average (regression) or vote (classification). No explicit learning — stores all training data. Fast training, slow prediction. Good for small datasets (< 100k samples); curse of dimensionality in high dimensions.

📏 Distance Metrics

Euclidean (L2): sqrt(Σ(x_i - y_i)²) — most common. Manhattan (L1): Σ|x_i - y_i| — robust to outliers. Cosine: angle between vectors — used in text and embedding search. Always scale features to make distances comparable.

🔍 Curse of Dimensionality

In high dimensions, all points are roughly equidistant — KNN breaks down. Use dimensionality reduction (PCA) or feature selection before KNN on high-D data. Rule of thumb: KNN works best when n_features < 20 and n_samples is moderate.

⚙️ Hyperparameter K

K=1 (nearest neighbour) = low bias, high variance. Large K = high bias, low variance. Test K from 3 to 30 on validation set. Odd K for classification (avoids ties). Larger K helps with imbalanced classes.

KEY TAKEAWAYS

  • Always scale inputs with StandardScaler — KNN is distance-based.
  • K=sqrt(n) is a rough starting heuristic; tune via cross-validation.
  • Cosine distance is your friend in embedding/RAG contexts (Azure AI Search uses it).
  • KNN is slow for prediction on large test sets — use tree-based methods for scale.
  • KNN naturally handles multi-class and multi-output problems.

Mini Project: Multi-Algorithm Comparison

4 hours

Why this matters: Apply all 5 algorithms (decision trees, random forests, XGBoost, SVM, KNN) to the same dataset. Compare on train/test metrics, training time, prediction time, and interpretability. This solidifies your understanding of when to use each algorithm and builds your model selection intuition.

📓 Day7_Mini_Project.ipynb

📊 Project Structure

Load a classification or regression dataset. Split into train/validation/test. For each algorithm, fit on train, tune on validation (via GridSearchCV), evaluate on test. Compute metrics, training time, prediction time, and feature importance.

🏆 Algorithm Comparison Framework

Speed: Trees >> SVM >> KNN on large data. Accuracy: Often XGBoost ≥ RF >> SVM > KNN > DT. Interpretability: DT >> RF >> SVM/KNN >> XGBoost. Scaling: Tree-based and KNN handle large datasets; SVM struggles. Choose based on data size, latency constraints, and interpretability needs.

🔧 Hyperparameter Tuning Best Practices

Use GridSearchCV for small parameter spaces (< 50 combinations). Use RandomizedSearchCV for large spaces. Always use stratified K-fold for classification (preserves class distribution). Set n_jobs=-1 for parallel training.

🎯 Deliverables

Create a comparison table (algorithm, best params, train score, test score, time). Write a 200-word analysis: which algorithm performed best? Why? Which would you use in production and why? Consider data size, interpretability, and latency.

20 Self-Assessment Flashcards

Score: 0/20
01 Decision Trees What is Gini impurity and how is it used in tree growth? ▼
Gini impurity = 1 - Σp² where p is the proportion of each class in a node. Lower Gini = purer node (more homogeneous classes). Trees grow by splitting on the feature that minimises weighted child Gini, greedily at each step. This greedy approach finds locally optimal splits, not necessarily globally optimal.
02 Decision Trees What do max_depth, min_samples_split, and min_samples_leaf control? ▼
max_depth = height of the tree (shallow = high bias, deep = high variance). min_samples_split = minimum samples required to split a node (high = fewer splits, higher bias). min_samples_leaf = minimum samples in a leaf node (acts as pruning to prevent overfitting). Tune these three together to control bias-variance trade-off.
03 Random Forests How does bootstrap aggregating (bagging) reduce overfitting? ▼
Bagging creates B bootstrap samples (sampling with replacement) and trains independent models on each. Predictions are averaged (regression) or voted (classification). If individual models are independent and high-variance, averaging reduces variance without increasing bias. Each bootstrap sample contains ~63% of original data; ~37% is held out (OOB), enabling free cross-validation.
04 Random Forests Why does random feature selection in RF decorrelate trees? ▼
If all trees use the same strong feature, their predictions are correlated, and averaging doesn't reduce variance as much. By limiting each split to sqrt(n_features) random features, weak features get a chance to be used, and trees are forced to find different patterns. Decorrelated trees → averaging reduces variance more → lower overall error.
05 Random Forests What is an Out-of-Bag (OOB) score and why is it useful? ▼
Each bootstrap sample excludes ~37% of data (OOB). Evaluate each tree on its OOB samples and average the scores. This gives you a validation score without needing a separate validation set. OOB score ≈ k-fold CV score with zero overhead. Use it to sanity-check generalization without splitting your data.
06 XGBoost Explain gradient boosting: how does it differ from bagging? ▼
Bagging trains independent trees in parallel, then averages. Boosting trains trees sequentially, where each new tree fits the residuals (errors) of the previous ensemble. New predictions = old predictions + (learning_rate × new tree). This sequential error-correction is why boosting often beats bagging, but without regularisation, boosting overfits faster.
07 XGBoost What role does eta (learning rate) play in XGBoost? ▼
eta (default 0.3) controls how much each tree contributes to predictions. Low eta (0.01–0.05) = slower learning, more trees needed, but often better generalisation. High eta = faster learning, fewer trees, risk of overfitting. Always pair low eta with early stopping to find the optimal number of boosting rounds.
08 XGBoost How does XGBoost handle missing values natively? ▼
XGBoost learns which direction (left or right) is better for missing values at each split during training. Each split has a default direction for missing values. This is why you don't need explicit imputation — XGBoost infers missingness patterns automatically. It's one of XGBoost's key advantages over sklearn's GradientBoosting.
09 SVM What is the maximum margin principle and why does it matter? ▼
SVM finds a hyperplane (decision boundary) that maximises the distance (margin) to the nearest points on either side. Wider margins generalise better to unseen data. Points closest to the margin are support vectors. This principle is motivated by statistical learning theory: wider margins → simpler model → better generalisation.
10 SVM Explain soft margins and the role of parameter C. ▼
Hard-margin SVM requires zero misclassification (often impossible). Soft-margin SVM allows some misclassification controlled by C (regularisation). High C = strict margins, penalise errors heavily (low bias, high variance). Low C = loose margins (high bias, low variance). Tune C via cross-validation.
11 SVM When would you use RBF kernel vs linear kernel in SVM? ▼
Linear kernel: when data is linearly separable or feature space is already high-dimensional. RBF kernel: when data is non-linear. RBF maps to infinite dimensions; good for most cases. Start with RBF and tune gamma (RBF kernel width). Polynomial for specific cases where polynomial boundaries make sense.
12 SVM Why must you scale SVM inputs? What scaler should you use? ▼
SVM is distance-based (margin is geometric distance). Unscaled features = features with larger ranges dominate the margin. Use StandardScaler to scale to mean=0, std=1. Always fit scaler on train data, then apply to validation/test. SVM is sensitive to scaling — this is a common mistake and source of poor performance.
13 KNN How does the curse of dimensionality affect KNN? ▼
In high dimensions, all points are roughly equidistant from each other (Euclidean distance becomes meaningless). KNN loses predictive power because "nearest" doesn't correspond to similar samples. Solution: reduce dimensionality with PCA or feature selection before applying KNN. KNN works best in < 20 dimensions.
14 KNN How do you choose K? What are the trade-offs? ▼
K=1 = nearest neighbour (low bias, high variance, overfits). Large K = high bias, low variance (underfits). Heuristic: start with K=sqrt(n). For classification, use odd K (avoids ties). Tune K via cross-validation on validation set. Larger K helps with imbalanced classes.
15 Distance Metrics Compare Euclidean, Manhattan, and cosine distance. ▼
Euclidean (L2): sqrt(Σ(x-y)²) — most common, sensitive to outliers. Manhattan (L1): Σ|x-y| — robust to outliers, computationally faster. Cosine: 1 - (x·y)/(|x||y|) — angle-based, used for embeddings and text. Use cosine for embeddings/RAG (Azure AI Search), Euclidean for general cases.
16 General When would you choose tree-based models vs SVM vs KNN? ▼
Tree-based (RF, XGBoost): fast, parallel, good with mixed data types. SVM: interpretable, best for small-medium datasets, needs scaling. KNN: simple, good for small data, non-parametric. In competitions: XGBoost wins. In production: trees for speed. In high-stakes applications: SVM for interpretability.
17 Hyperparameter Tuning What is GridSearchCV and when do you use RandomizedSearchCV instead? ▼
GridSearchCV: exhaustive search over all combinations (best for small parameter spaces, < 50 combinations). RandomizedSearchCV: sample random combinations (best for large spaces). RandomizedSearchCV often finds good solutions faster and with less compute. Use stratified k-fold for classification to preserve class distribution.
18 Model Evaluation Explain train/validation/test split. Why not just train/test? ▼
Train: fit model. Validation: tune hyperparameters and select model. Test: final evaluation on held-out data. If you tune on test set, you overfit to it. Proper split: 60% train, 20% validation, 20% test. For hyperparameter tuning, use k-fold CV on train+val combined, then final evaluation on test.
19 Feature Engineering How do tree-based models and distance-based models handle features differently? ▼
Trees don't need scaling or one-hot encoding (handle categorical natively). Distance-based (SVM, KNN) are scale-sensitive — use StandardScaler. Trees are invariant to monotonic transformations (e.g., log). Distance methods depend on raw values. This is why feature engineering for trees is minimal; for SVM/KNN, it's critical.
20 Model Selection You have 3 candidate models with test scores 0.85, 0.86, 0.87. How do you choose? ▼
Don't just pick the highest score. Check: (1) training time and prediction latency, (2) interpretability needs, (3) statistical significance (confidence intervals), (4) robustness to data distribution shift. Often, the simplest/fastest model is best. If differences < 1%, choose for speed and interpretability, not raw accuracy.
